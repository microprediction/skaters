Markdown skills you can point Claude (or Claude Code) at — for using skaters, for reviewing forecasting code, and for benchmarking any time-series paper that ships code.

A skill here is just a markdown file of instructions. Drop it in your repo, or paste it into a chat, and Claude follows the procedure. Each one below lives in the skaters repository.

benchmark-review — for reviewing time-series papers

A reviewer's procedure for any paper claiming univariate distributional prediction with code. The core rule: every method — theirs, yours, and the baselines — is turned into the same predictive object and scored by the same code on the same held-out points. The skill walks through:

  • Classify the output — density, samples, quantiles, or a CDF — which decides whether it can be log-likelihood-scored at all (a conformal CDF cannot).
  • Fix one target and one protocol — one-step change, rolling refit (or zero-shot for foundation models); never mix protocols in one table.
  • Bias-free universe — a fixed selection rule, continuous vs repeat-heavy split, and per-series win-rates (family-clustering inflates).
  • Adversarial checklist — toughest baselines not straw men, constant mean model across wrapped comparisons, reconstruction caveats, reproducible from one script.

skaters ships the reference harness it describes: benchmarks/sota_study.py (nine rolling baselines incl. Prophet) and foundation_study.py (zero-shot Chronos / TimesFM / Moirai / Lag-Llama). Read the skill →
